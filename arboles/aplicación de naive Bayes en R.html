Clasificación con Naive Bayes en R

Guía Práctica de Implementación usando el Dataset Iris

El clasificador de Naive Bayes es un algoritmo basado en el teorema de Bayes que asume independencia condicional entre las variables predictoras dado el valor de la variable objetivo. Es especialmente útil por su eficiencia y velocidad en problemas de clasificación.

1. Instalación y Carga de Librerías

Para este análisis requerimos el paquete e1071 (que contiene el algoritmo Naive Bayes) y el paquete caret (para generar la matriz de confusión y evaluar el modelo).

# Instalar si no los tienes instalados previamente
install.packages("e1071")
install.packages("caret")

# Cargar las librerías en tu script de R
library(e1071)
library(caret)

2. Preparación y División de Datos

Es fundamental dividir el conjunto de datos iris en un 80% para entrenamiento (guardado en la variable datos_entrenamiento) y un 20% para prueba (guardado en datos_prueba).

# Fijar una semilla para reproducibilidad
set.seed(123)

# Generar índices aleatorios para la división
indices <- sample(1:nrow(iris), size = 0.8 * nrow(iris))

# Crear los subconjuntos correspondientes
datos_entrenamiento <- iris[indices, ]
datos_prueba        <- iris[-indices, ]

3. Entrenamiento del Modelo

Entrenamos el clasificador utilizando la función naiveBayes(). La estructura de la fórmula Species ~ . le indica a R que use todas las variables del dataset (Longitud/Ancho de Sépalo y Pétalo) para predecir la especie de la planta.

# Entrenar el clasificador Naive Bayes
modelo_nb <- naiveBayes(Species ~ ., data = datos_entrenamiento)

# Visualizar el resumen de probabilidades condicionales calculadas
print(modelo_nb)

Nota Matemática: Como las variables de iris son numéricas continuas, el algoritmo calcula la media y la desviación estándar para cada clase asumiendo por defecto una distribución Gaussiana (Normal).

4. Realizar Predicciones

Una vez entrenado el algoritmo, procedemos a clasificar las flores del conjunto de prueba, el cual se mantuvo oculto para el modelo durante la etapa anterior.

# Predecir las clases del conjunto de datos de prueba
predicciones <- predict(modelo_nb, newdata = datos_prueba)

# Mostrar las primeras predicciones resultantes
head(predicciones)

5. Evaluación Mediante Matriz de Confusión

Finalmente, comparamos el rendimiento de las predicciones del algoritmo frente a las etiquetas reales del dataset botánico mediante una matriz de confusión.

# Generar la matriz de confusión detallada
resultado_evaluacion <- confusionMatrix(predicciones, datos_prueba$Species)

# Imprimir el reporte completo de métricas
print(resultado_evaluacion)

Métricas Clave a Evaluar

  • Accuracy (Exactitud): Porcentaje total de plantas clasificadas de forma correcta (suele superar el 93% en este conjunto de datos).
  • Sensibilidad (Sensitivity): Capacidad del algoritmo para detectar correctamente una especie específica sin pasarla por alto.
